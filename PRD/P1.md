# P1: UI/UX Design & Visual Style

기준일: 2026-03-26

## 목적

P1은 ABA Edu Portal의 화면 구현 체크리스트 문서가 아니라, 실제 제품 화면을 설계할 때 따라야 하는 UI/UX 디자인 기준 문서다.

이 문서는 아래 두 레퍼런스를 기반으로 한다.

- 레이아웃/콘텐츠 구조 레퍼런스: `https://claude.com/blog`
- 시각 스타일 레퍼런스: `https://codewiki.google` 라이트 모드

목표는 다음과 같다.

- 콘텐츠를 빠르게 읽고 탐색할 수 있는 content-first UI
- 과장된 장식보다 읽기 편한 여백, 타이포, 정보 구조를 우선하는 minimal UI
- 부모 사용자가 모바일에서 부담 없이 스캔하고, 필요한 교육 자료에 곧바로 접근할 수 있는 UX

## Experience Keywords

- Simple
- Minimal
- Editorial
- Content-friendly
- Calm
- Trustworthy
- Warm but not playful
- Lightweight

## Design Direction

### 1. Layout Direction

`claude.com/blog`의 장점을 참고해 아래 원칙을 따른다.

- 첫 화면에서 서비스 목적과 주요 탐색 동선을 즉시 이해할 수 있어야 한다
- 상단은 간결한 header, 그 아래는 큰 제목과 짧은 설명, 이후 카테고리/필터/콘텐츠 리스트 순으로 구성한다
- 콘텐츠 자체가 주인공이어야 하며, 카드 장식보다 제목, 요약, 메타 정보, 카테고리 구분이 우선한다
- 페이지는 section이 많더라도 시각적으로 조용해야 하며, 각 section은 충분한 vertical spacing으로 분리한다
- 사용자는 홈에서 읽을거리와 도구를 동시에 발견할 수 있어야 하지만, 정보 우선순위는 항상 콘텐츠가 먼저다

### 2. Visual Style Direction

`codewiki.google` 라이트 모드의 장점을 참고해 아래 원칙을 따른다.

- 전체 배경은 밝고 깨끗한 off-white 계열을 사용한다
- 카드와 패널은 흰색 또는 매우 연한 neutral surface 위주로 구성한다
- 그림자보다 border와 subtle contrast로 레이어를 구분한다
- 색은 브랜드 강조용이 아니라 정보 구획과 CTA 강조용으로 절제해서 사용한다
- 전체 인상은 “친절한 교육 서비스”여야 하며 “어린이용 장난감 사이트”처럼 보이면 안 된다

## UI/UX Principles

### Content First

- 홈, 카테고리, 상세 페이지 모두 headline, summary, metadata, category 정보가 가장 먼저 읽혀야 한다
- 썸네일은 보조 요소이며, 텍스트 계층을 방해하지 않아야 한다
- 카드 내부 정보량은 많아도 되지만 구조는 단순해야 한다

### Low Cognitive Load

- 한 화면에서 동시에 강조되는 CTA는 1개만 둔다
- 사용자는 “무엇을 읽을지”, “어떤 앱을 써볼지”, “어디서 다시 탐색할지”를 바로 이해할 수 있어야 한다
- 필터, 정렬, 보기 전환은 노출하되 복잡한 조작으로 느껴지지 않아야 한다

### Editorial Rhythm

- section 간 간격은 넉넉하게 가져간다
- 긴 텍스트는 narrow content width를 유지해 읽기 쉽게 만든다
- list/grid 전환이 가능하더라도 기본값은 읽기 쉬운 list 또는 2-column editorial grid를 우선 검토한다

### Mobile Practicality

- 모바일에서 thumb zone 안에 주요 CTA가 들어오도록 설계한다
- 카드, 필터 chip, 버튼은 최소 `44x44px` 터치 영역을 확보한다
- hover에 의존하지 않고 tap 기준으로 인터랙션을 설계한다

## Information Architecture

## Global Navigation

- 좌측: 로고
- 우측: 핵심 메뉴 3개 이내
- 추천 메뉴 구조:
  - 콘텐츠
  - 교육 앱
  - 소개

보조 탐색 요소:

- 검색 또는 빠른 탐색 입력
- 모바일에서는 drawer/menu로 축약

## Global Page Structure

모든 주요 페이지는 아래 리듬을 공유한다.

1. `Header`
2. `Page Intro`
3. `Primary Navigation or Filter`
4. `Main Content Area`
5. `Secondary Discovery Section`
6. `Footer`

## Page Guidelines

## 1. Home

### 목적

서비스의 성격을 설명하고, 사용자가 바로 읽기 시작하거나 앱으로 이동할 수 있게 한다.

### 구조

1. Hero
2. Featured Categories / Topic shortcuts
3. Latest or Featured Content feed
4. Parent Guide section
5. Education Apps preview
6. Final CTA or reassurance section

### UX 원칙

- Hero는 짧고 명확해야 한다
- 긴 설명보다 “무엇을 얻는지”를 한두 문장으로 설명한다
- 첫 fold 안에서 콘텐츠 진입 CTA가 보여야 한다
- 앱 섹션은 보조 섹션으로 두되, 시각적으로 과도하게 튀지 않게 한다

## 2. Category Page

### 목적

주제별 콘텐츠를 탐색하고 필터링하는 browsing page다.

### 구조

1. Category title + short description
2. Type filter chips
3. Result count / sorting / optional view toggle
4. Content list or grid
5. Empty state or related category links

### UX 원칙

- `claude.com/blog`처럼 제목과 메타 정보 중심의 브라우징 경험을 제공한다
- 기본값은 한눈에 제목과 요약을 읽기 쉬운 layout이어야 한다
- filter chip은 선택 상태가 명확해야 하며, 1 tap으로 on/off 가능해야 한다

## 3. Content Detail

### 목적

학습 콘텐츠를 집중해서 읽게 만드는 reading page다.

### 구조

1. Breadcrumb or back link
2. Content title
3. Category / content type / reading time / date
4. Hero image or media
5. Main article body
6. Inline callout / key takeaway
7. Related content

### UX 원칙

- 본문 가독성이 가장 중요하다
- body container는 너무 넓지 않게 제한한다
- 이미지, 영상, 카드뉴스 시퀀스는 본문 리듬을 깨지 않게 삽입한다
- 관련 콘텐츠는 본문 하단에서 자연스럽게 이어지게 한다

## 4. Apps List

### 목적

교육 앱을 탐색하고 실행 가능한 경험으로 연결한다.

### 구조

1. Page intro
2. 앱 분류 또는 상태 구분
3. App cards
4. Live / Coming Soon 구분

### UX 원칙

- 앱 카드는 콘텐츠 카드보다 조금 더 product-card 성격을 가진다
- 하지만 전체 사이트 톤은 유지해야 하므로 과한 dashboard 스타일은 피한다
- 실사용 가능 여부가 시각적으로 즉시 구분되어야 한다

## 5. App Detail

### 목적

앱 설명, 학습 목적, 사용 연령, 실행 CTA를 분명히 전달한다.

### 구조

1. Back link
2. App title + short summary
3. App meta
4. Preview screenshot or frame
5. Primary CTA
6. How to use
7. Related apps or related content

### UX 원칙

- 좌우 분할 레이아웃을 쓰더라도 모바일에서는 단일 컬럼으로 자연스럽게 접혀야 한다
- CTA는 “앱 실행” 하나가 명확히 보여야 한다
- 설명 문구는 부모 관점에서 실용적으로 작성한다

## 6. About

### 목적

서비스 신뢰도를 높이고 운영 목적을 설명한다.

### 구조

1. Mission statement
2. Why this service exists
3. 운영 원칙 또는 콘텐츠 철학
4. Contact

### UX 원칙

- 장식적인 branding page가 아니라 신뢰 기반의 information page여야 한다
- 텍스트 중심 구성이 적합하다

## 7. 404 / Empty State

### 목적

사용자를 막히게 하지 않고 다음 행동으로 다시 연결한다.

### UX 원칙

- 친절하고 짧은 메시지
- 홈으로 이동, 콘텐츠 보기, 앱 보기 중 1~2개 CTA 제공
- 일러스트는 optional이며 필수는 아니다

## Visual System

## Color System

전체 톤은 라이트 모드 전용으로 정의한다.

### Base Colors

- `bg.canvas`: `#F7F7F4`
- `bg.surface`: `#FFFFFF`
- `bg.subtle`: `#F1F3EF`
- `border.default`: `#D9DED6`
- `border.strong`: `#B9C1B5`
- `text.primary`: `#1F241F`
- `text.secondary`: `#5E665E`
- `text.tertiary`: `#7A837A`

### Accent Colors

- `accent.primary`: `#2F6B4F`
- `accent.primary-soft`: `#E5F1EA`
- `accent.secondary`: `#6A7E99`
- `accent.secondary-soft`: `#EEF3F8`
- `accent.warm`: `#C88A3D`
- `accent.warm-soft`: `#F9EFE3`

### Usage Rules

- 기본 페이지는 `bg.canvas`를 사용한다
- 카드와 본문 박스는 `bg.surface`를 사용한다
- 강조 배경은 매우 옅은 tint만 사용한다
- raw hex를 컴포넌트에 직접 박지 않고 semantic token으로 관리한다
- 강한 그라디언트, neon 계열, candy-like pastel 조합은 사용하지 않는다

## Typography

### Font Direction

- Heading: readable serif 또는 humanist sans
- Body: neutral sans

추천 조합:

- Heading: `Pretendard` 또는 `Source Serif 4`
- Body/UI: `Pretendard` 또는 `Inter`

### Typography Rules

- `h1`: 40-48px / line-height 1.15
- `h2`: 28-32px / line-height 1.2
- `h3`: 22-24px / line-height 1.3
- `body-lg`: 18px / line-height 1.7
- `body`: 16px / line-height 1.7
- `caption`: 14px / line-height 1.5
- 본문은 mobile에서도 최소 16px 유지
- 긴 텍스트는 60-75 characters line length를 넘기지 않도록 제어

## Spacing

- spacing scale은 8px rhythm을 사용한다
- section vertical padding은 `64 / 80 / 96px` tier를 기본으로 한다
- card internal padding은 `20 / 24 / 28px` 범위에서 통일한다
- chip, badge, meta row는 너무 조밀하지 않게 `8-12px` gap을 유지한다

## Radius / Border / Shadow

- 기본 radius: `16px`
- large card radius: `20px`
- pill/chip radius: `999px`
- 기본 shadow는 거의 쓰지 않거나 매우 약하게 사용한다
- 레이어 구분은 border와 background contrast를 우선 사용한다

권장 스타일:

- `border: 1px solid var(--border-default)`
- `shadow: 0 1px 2px rgba(16, 24, 16, 0.04)` 수준

## Component Style

## Buttons

### Primary Button

- solid accent color
- medium weight text
- radius 999px 또는 14px
- hover보다 active/pressed 상태가 더 중요

### Secondary Button

- white or subtle surface
- border visible
- text-primary 사용

### Button Rules

- 한 section 안의 primary CTA는 1개만
- 텍스트는 동사형으로 명확하게 작성
- 아이콘이 있더라도 label을 대체하지 않는다

## Cards

### Content Card

- 제목, 요약, 메타 정보 순
- 썸네일은 optional
- 카드 높이를 억지로 맞추기보다 읽기 리듬을 우선

### App Card

- 제품 preview, 상태 badge, 핵심 기능 1-2개, CTA
- `Live`, `Coming Soon` 상태가 즉시 보여야 함

## Chips / Tags

- 카테고리와 콘텐츠 타입 구분에 사용
- 선택 상태는 fill 또는 border contrast로 명확히 표현
- decorative chip 남용 금지

## Search / Filter

- `claude.com/blog`처럼 조용한 입력창 스타일
- border가 분명하고 placeholder 대비가 충분해야 함
- 검색과 필터는 main content보다 더 강하게 보이면 안 된다

## Motion

- motion은 subtle하게만 사용
- hover lift, scale, fade 정도로 제한
- duration은 `150-220ms`
- reduced motion 환경에서는 대부분 제거
- 콘텐츠 페이지에서는 flashy animation 사용 금지

## Responsive Strategy

### Breakpoints

- mobile: `0-639`
- tablet: `640-1023`
- desktop: `1024-1439`
- wide: `1440+`

### Layout Rules

- 모바일 우선 설계
- 홈 피드:
  - mobile 1열
  - tablet 2열
  - desktop 2-3열
- 콘텐츠 상세 본문:
  - mobile 단일 컬럼
  - desktop에서도 본문 영역은 narrow width 유지
- 앱 상세:
  - mobile 단일 컬럼
  - desktop 2-column 가능

## Accessibility

- body text contrast `4.5:1` 이상
- icon-only button 금지, unavoidable하면 `aria-label` 필수
- heading hierarchy 유지
- 모든 CTA는 keyboard focus visible 상태 제공
- touch target 최소 `44x44px`
- color만으로 상태를 구분하지 않는다
- 오류, 빈 상태, 비활성 상태는 텍스트 설명을 함께 제공한다

## Copy Style

- 짧고 명확한 한국어
- 전문 용어는 필요한 만큼만 사용
- 부모가 바로 이해할 수 있는 생활형 문장 선호
- “멋있어 보이는 문구”보다 “무엇을 도와주는지”를 먼저 말한다

## Anti-Patterns

아래 스타일은 이번 프로젝트에서 지양한다.

- 과한 pastel playground 톤
- 두꺼운 shadow와 glossy card
- 과장된 gradient background
- 화면마다 제각각 다른 색상 체계
- 내용보다 썸네일이 더 크게 보이는 카드
- 모바일에서 14px 이하 본문
- 너무 많은 배지, 아이콘, 강조 컬러
- blog와 app 영역이 전혀 다른 제품처럼 보이는 분절감

## Done When

- 홈, 카테고리, 상세, 앱 관련 페이지가 모두 동일한 design language를 공유한다
- 콘텐츠 탐색 흐름이 `Home -> Category -> Content Detail`로 자연스럽게 이어진다
- 모바일에서 제목, 요약, CTA를 첫 스크린 안에서 이해할 수 있다
- 라이트 모드 기준으로 전체 서비스가 밝고 차분하며 읽기 쉬운 인상을 준다
- 콘텐츠 카드와 앱 카드가 같은 브랜드 안에서 보이되 역할 차이는 분명하다
- 본문 가독성이 확보되고, 줄 길이와 여백이 안정적이다
- 필터, 검색, 보기 전환 요소가 본문을 방해하지 않는다
- 320px부터 1440px+까지 horizontal scroll 없이 동작한다
- 모든 인터랙티브 요소가 터치 영역과 focus state를 만족한다

## Notes

- 본 P1은 기존의 playful/warm 중심 시안에서 한 단계 더 절제된 editorial 방향으로 재정의한 문서다
- 이후 구현 시 색상, spacing, typography token은 `globals.css` 또는 design token 파일로 승격하는 것을 권장한다
- 다크 모드는 P0 범위 밖이므로 이번 문서에서는 정의하지 않는다
