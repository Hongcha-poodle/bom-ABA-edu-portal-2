# ABA Edu Portal 홈 개편 작업 문서

## 프로젝트 목적

현재 홈 화면은 방향은 좋지만, 같은 메시지가 여러 섹션에서 반복되고 있고 설명형 카피가 많아서 실제 사용자의 행동 흐름이 약해져 있다.

목표는 다음과 같다.

* 중복되는 카피와 섹션 역할 제거
* 설명보다 선택과 탐색 중심 구조로 재정렬
* 부모 사용자가 빠르게 이해하고 바로 이동할 수 있는 단순한 UX로 정리
* 홈의 핵심 흐름을 `주제 선택 → 콘텐츠 보기 → 앱 보기`로 명확화

---

# 세션 1 — 홈 정보 구조 재정렬

## 목표

홈의 전체 섹션 순서를 단순화하고, 사용자의 첫 행동이 더 빨리 일어나도록 구조를 재배치한다.

## 현재 문제

* 히어로 아래에 설명형 섹션이 먼저 나와서 실제 행동 유도가 늦다
* “오늘의 시작”, “부모 가이드”, “맞춤 추천”이 주제 탐색보다 앞뒤로 섞여 있어 흐름이 분산된다
* 사이트의 핵심 진입점은 주제 선택인데, 현재 우선순위가 충분히 높지 않다

## 작업 지시

홈 섹션 순서를 아래처럼 재배치한다.

### 변경 전 개념

* 헤더
* 히어로
* 오늘의 시작
* 맞춤 추천
* 주제 바로가기
* 추천 콘텐츠
* 부모 가이드
* 교육 앱
* 푸터

### 변경 후 개념

* 헤더
* 히어로
* 주제 선택
* 추천 콘텐츠
* 교육 앱
* 짧은 소개
* 푸터

## 구현 원칙

* 홈에서는 설명보다 선택 UI가 먼저 보여야 한다
* 중복 역할 섹션은 뒤로 미루지 말고 삭제 또는 통합한다
* 첫 스크롤 구간 내에 “내가 어디로 가야 하는지”가 드러나야 한다

## 완료 조건

* 홈 구조가 위 순서대로 정리되어 있어야 함
* 주제 선택 섹션이 히어로 바로 아래에 위치해야 함
* 설명형 섹션 수가 줄어들어야 함

---

# 세션 2 — 히어로 카피와 CTA 단순화

## 목표

히어로를 설명 블록이 아니라, 가치 제안과 행동 유도 중심 영역으로 정리한다.

## 현재 문제

* 문장이 다소 많고, 아래 섹션과 의미가 겹친다
* “빠르게”, “실천”, “앱 연결” 메시지가 이후 섹션에서도 반복된다
* 히어로 하단 보조 문구가 길어 첫 인상이 무거워진다

## 작업 지시

### 히어로에서 남길 것

* 사이트의 핵심 가치 제안
* CTA 2개
* 부모 중심 ABA 실천 포털이라는 정체성

### 히어로에서 줄일 것

* 장문 설명
* 반복형 보조 카피
* 기능 나열형 문장

## 권장 카피 예시

### 제목

아이와 바로 해볼 ABA 실천 콘텐츠

### 설명

언어, 사회성, 생활 루틴, 행동 관리를 부모가 쉽게 이해하고 적용할 수 있게 정리한 포털

### 버튼

* 주제 선택하기
* 교육 앱 보기

## 구현 원칙

* 히어로는 한눈에 이해되어야 한다
* 한 문장 안에 메시지를 여러 개 넣지 않는다
* CTA 문구는 짧고 행동 중심으로 유지한다

## 완료 조건

* 히어로 문장 수가 줄어들어야 함
* CTA가 명확히 2개로 정리되어 있어야 함
* 아래 섹션과 의미 중복이 최소화되어야 함

---

# 세션 3 — “오늘의 시작” 섹션 제거 또는 축소

## 목표

설명형 사용 가이드를 별도 대형 섹션으로 두지 않고, 더 간단한 형태로 정리한다.

## 현재 문제

* “고민 선택 → 짧게 읽기 → 앱으로 이동” 구조는 좋지만, 이건 홈 구조 자체가 보여줘야 할 흐름이다
* 별도 섹션으로 존재하면 설명이 중복된다
* 히어로와 부모 가이드 사이에서 역할이 애매해진다

## 작업 지시

아래 두 가지 중 하나로 처리한다.

### 권장안 A

섹션 자체 삭제

### 대안 B

히어로 아래 작은 3-step 텍스트로 축소

* 고민 선택
* 가이드 보기
* 앱으로 연습하기

## 구현 원칙

* 홈의 구조가 설명을 대신해야 한다
* 동일한 메시지를 섹션으로 반복하지 않는다

## 완료 조건

* 현재 “오늘의 시작” 대형 섹션이 제거되거나 매우 짧은 보조 UI로 축소되어야 함
* 사용자 흐름 설명이 더 이상 장문 블록으로 남아 있지 않아야 함

---

# 세션 4 — “맞춤 추천”과 “주제 바로가기” 통합

## 목표

탐색 시작점을 하나로 정리해서 사용자가 고민 없이 선택할 수 있게 만든다.

## 현재 문제

* 맞춤 추천과 주제 바로가기 모두 “무엇을 볼지 선택”하는 역할을 한다
* 추천과 탐색이 분리되어 보여 구조가 분산된다
* 일부 주제가 추천 카드와 아래 콘텐츠 카드에서 반복된다

## 작업 지시

“맞춤 추천” 섹션을 없애고, “주제 바로가기” 섹션 하나로 통합한다.

## 권장 섹션 제목

지금 필요한 주제를 선택해

## 권장 카드 예시

* 말문 트기
  요청하기, 따라 말하기, 첫 언어 표현
* 사회성
  차례 지키기, 기다리기, 함께 놀이
* 생활 루틴
  식사, 옷 입기, 정리, 양치
* 행동 이해
  행동의 이유 관찰, 대체행동 지도
* ABA 기초
  강화, 프롬프트, 실천 원리

## 구현 원칙

* 같은 역할의 탐색 섹션은 하나만 둔다
* 카드 문장은 짧아야 한다
* 카드 전체 클릭이 가능하게 하고, 내부 CTA 문구는 최소화한다

## 완료 조건

* 맞춤 추천 섹션이 제거되어야 함
* 주제 탐색 시작점이 하나로 통합되어야 함
* 사용자가 첫 선택을 더 빠르게 할 수 있어야 함

---

# 세션 5 — 추천 콘텐츠 카드 압축

## 목표

홈의 콘텐츠 카드를 요약형으로 바꿔 스캔 속도를 높인다.

## 현재 문제

* 카드 안 정보가 많아 홈에서 읽기 부담이 있다
* 카테고리, 설명, 형식, 난이도, 부가 문구가 동시에 보여 시선이 분산된다
* 홈은 탐색용인데 상세 정보가 너무 많이 들어와 있다

## 작업 지시

콘텐츠 카드를 아래 구조로 단순화한다.

## 권장 카드 구조

* 태그 2~3개
  예: 언어 / 2–5세 / 5분
* 제목
* 한 줄 설명
* CTA: 보기

## 제거 권장 요소

* 장문 설명
* 중복되는 형식 설명
* 홈에서 필요 없는 상세 메타 정보
* 반복형 CTA 문구

## 예시

### 선택 보드로 요청하기 시작하기

그림을 짚으며 원하는 것을 표현하는 첫 요청 연습
[보기]

### 차례 기다리기 첫 연습

놀이 상황에서 3초 기다리기부터 시작하는 간단한 루틴
[보기]

## 구현 원칙

* 홈 카드의 역할은 “이 콘텐츠를 눌러볼지 판단하게 하는 것”까지다
* 자세한 정보는 상세 페이지로 넘긴다

## 완료 조건

* 카드당 텍스트 길이가 줄어들어야 함
* 한눈에 스캔 가능해야 함
* 첫 화면에서 콘텐츠 여러 개를 빠르게 비교할 수 있어야 함

---

# 세션 6 — 부모 가이드 섹션 축소 또는 제거

## 목표

부모 가이드 섹션의 설명 반복을 줄이고, 홈의 밀도를 낮춘다.

## 현재 문제

* “짧게 읽기”, “바로 시도”, “한 가지 목표” 메시지가 이미 다른 섹션과 겹친다
* 좋은 내용이지만 홈에 대형 섹션으로 있을 필요는 낮다
* 읽는 양만 늘어나고, 실제 이동 유도는 약하다

## 작업 지시

기존 부모 가이드 섹션은 삭제하거나, 짧은 소개 문단 하나로 축소한다.

## 축소안 예시

부모가 쉽게 이해하고 바로 시도할 수 있도록, 콘텐츠와 앱을 한 흐름으로 연결한 실천형 포털

## 구현 원칙

* 홈은 서비스 소개 페이지가 아니라 탐색 시작 페이지다
* 소개 문구는 한 번만, 짧게 보여준다

## 완료 조건

* 부모 가이드의 3블록 구조가 제거되어야 함
* 같은 의미의 설명이 반복되지 않아야 함

---

# 세션 7 — 교육 앱 섹션 요약형 리디자인

## 목표

앱 카드도 콘텐츠 카드처럼 간결하게 바꿔 홈 전체 톤을 통일한다.

## 현재 문제

* 앱 기능 설명이 길어 홈에서 읽기 부담이 있다
* bullet 정보가 많아 홈보다 상세 소개 영역처럼 보인다
* 콘텐츠 섹션과 비교했을 때 정보량 밸런스가 맞지 않는다

## 작업 지시

앱 카드를 요약형 카드 구조로 변경한다.

## 권장 카드 구조

* 상태 배지
  예: Live / Coming Soon
* 앱명
* 한 줄 설명
* 연령 또는 사용 시간
* CTA

## 예시

### 시아 한글 키보드

큰 버튼과 소리로 한글에 친해지는 첫 연습
4세 이상 · 5–10분
[앱 보기]

### 냠냠 쩝쩝

놀면서 음식 단어와 분류를 익히는 짧은 활동
3–7세 · 3–7분
[앱 보기]

## 구현 원칙

* 홈에서는 “무슨 앱인지”만 이해되면 충분하다
* 상세 기능은 앱 상세 페이지에서 보여준다

## 완료 조건

* 앱 카드 텍스트 길이가 줄어들어야 함
* 콘텐츠 섹션과 시각적 밀도가 유사해야 함
* 콘텐츠 → 앱 흐름이 자연스러워야 함

---

# 세션 8 — 카피 전체 정리 및 반복 표현 제거

## 목표

홈 전반의 카피를 정리해 메시지 반복을 줄이고, 더 또렷한 톤으로 통일한다.

## 현재 문제

아래 표현들이 여러 번 반복된다.

* 빠르게
* 바로
* 5분
* 실천
* 자연스럽게 연결
* 복잡한 이론은 덜어내고

표현 자체는 나쁘지 않지만, 홈에서 너무 자주 나오면 설득력이 떨어지고 피로감이 생긴다.

## 작업 지시

### 원칙

* 한 섹션당 핵심 메시지 1개만
* 카드 설명은 한 줄
* CTA는 짧게 통일
* 같은 의미를 다른 섹션에서 반복하지 않기

### CTA 통일안

* 보기
* 시작하기
* 앱 보기
* 더 보기

### 줄여야 할 표현

* 오늘 바로
* 빠르게 읽고 바로
* 자연스럽게 이어가요
* 한눈에 쏙
* 지금 당장

## 구현 원칙

* 브랜드 톤은 부드럽게 유지하되 문장은 짧게
* 감성 카피보다 사용 맥락 전달이 우선

## 완료 조건

* 전체 홈 카피가 더 짧아져야 함
* 반복 단어 빈도가 줄어들어야 함
* 섹션마다 말하는 내용이 명확히 달라야 함

---

# 세션 9 — 최종 QA 체크

## 목표

개편 후 홈이 실제로 더 심플하고 직관적으로 느껴지는지 검증한다.

## 체크리스트

* 첫 화면에서 사용자가 어디를 눌러야 할지 바로 이해되는가
* 주제 선택이 가장 빠른 진입점으로 보이는가
* 설명형 섹션이 과도하게 남아 있지 않은가
* 콘텐츠 카드와 앱 카드가 같은 밀도로 정리되어 있는가
* 같은 메시지가 두 번 이상 반복되지 않는가
* 모바일에서 스캔 속도가 빨라졌는가
* 전체 페이지 길이가 이전보다 줄었는가

## 완료 조건

* 홈의 핵심 흐름이 `주제 선택 → 콘텐츠 → 앱`으로 명확히 보일 것
* 설명보다 행동 유도가 먼저 보일 것
* 중복 섹션이 정리되어 있을 것
